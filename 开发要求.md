# 开发要求
## 任务介绍
我在做安卓开发，并研究如何利用AI来复刻公开的京东APP，但我的复刻无需100%相似，我只需要做到如下几点：
- 1、UI结构基本相似，图标做不到一模一样不要紧，类似即可。
- 2、不需要做出APP的所有页面，例如京东APP，我们只做app需实现的任务用到的典型页面就可以了，其他的页面，比如小视频、秒送页面，我们全部设置为空页面，只保留一个入口给后续开发接入，暂时不需要处理。
- 3、不需要实现动态，例如不需要实现动态数据，商品列表、店铺信息等全部固定死。
- 4、不需要数据库，但是需要保存重要的信号（这些信号我会指定），如购物车商品、订单状态
- 5、不需要登录和联网，仅为本地演示使用
- 6、开发过程，可以考虑截图给AI，让AI实现开发任务。
- 7、没有任何限制，对框架等的选取完全取决于开发者习惯
- 8、不需要通知权限等复杂的东西，至多需要下保存图片到本地、读取相册这样的权限

## 任务要求
1. 开发的时候可以采取逐步开发的策略，慢慢补充需求，慢慢新增数据字段等
2. 避免单个文件代码过长，**必须要遵循MVP模式**
3. 按照指定的页面结构、参考给定的截图作为UI设计指导
4. 严格按照已经定义好的数据结构`app\src\main\java\com\example\MyJD\model`和已有的数据`app\src\main\assets\data`
5. 按照用户要求进行开发，除非用户指定，否则不要引入新的内容。
    - 你只需要参考我给你的截图，**不要脑补**截图中各个按钮点击后会达到的效果。
    - ***截图里面的信息（例如人的名字、人的头像）不必参考***，因为我们自己预设了已有的数据`app\src\main\assets\data`
6. UI中，各个图标若找不到合适的，可以用emoji表情代替。但优先考虑`androidx.compose.material.icons.Icons, androidx.compose.material.icons.filled.AccountCircle`等
7. 严禁执行删除文件等危险操作。
8. 每当你修改完代码，记得更新`README.md`文件，并且确定build正常了再交付给我。禁止修改`需求.md`.
9. 如果你需要图片支持，请给我反馈，告诉我需要什么图片。如果你使用了我给你的图片，记得把图片名改为一个可读的名字，便于后期维护。

## 你当前的任务
设计 MeTab 页面，该页面的要素包括：

- **顶部信息区（UserHeader）**  
  展示用户头像、昵称、会员等级及快捷入口。
  - 头像：圆形图片或占位符（emoji可替代）。
  - 用户名：静态展示，如“Ddddddjy”。
  - 会员标签：如“铜牌会员”、“学生优惠”等。
  - 快捷入口：包括“客服”、“地址”、“设置”图标。
  - 背景颜色使用红色渐变，突出品牌主色。

- **会员权益区（MemberSection）**  
  模拟京东会员中心样式，展示优惠券、京豆、红包、白条取现、外卖等静态模块。
  - 每个功能块使用 `Row` 布局，文字说明与图标垂直排列。
  - “更多”按钮可跳转至空页面或Toast提示。

- **活动横幅区（PromoBanner）**  
  展示促销活动，如“品类补贴券”、“11.11抢先购”等。
  - 采用 `Row` + `Card` 布局，每个卡片包含文字与图片。
  - 点击后无需实际跳转，可保留点击反馈。

- **订单功能区（OrderSection）**  
  标题：“我的订单”，右侧带“查看全部”文字按钮。
  - 使用 `Row` 展示订单状态图标：“待付款”、“待收货”、“待使用”、“待评价”、“退换/售后”。
  - 每个状态为可点击卡片，点击进入空页面。

- **资产与服务区（AssetServiceSection）**  
  分为两列布局：
  - 左侧“钱包”展示余额、白条金额、金条、京东黄金等静态数值。
  - 右侧“服务家”展示服务类项目，如“京东快递”、“在线医生”等。

- **互动与推荐区（InteractionSection）**  
  展示互动内容和营销活动，如“打卡领京豆”、“天天领豆”、“东东农场”等。  
  每个项目为静态卡片，点击后无实际跳转。

- **底部导航栏（BottomNavigation）**  
  与其他页面一致，包含：
  - 首页（`Icons.Filled.Home`）
  - 视频（`Icons.Filled.PlayCircle`）
  - 消息（`Icons.Filled.Chat`）
  - 购物车（`Icons.Filled.ShoppingCart`）
  - 我的（`Icons.Filled.Person`，当前高亮）


### 页面功能说明
- 页面静态展示用户信息与功能入口。
- 点击“设置”图标 → 跳转至设置页（SettingActivity）。
- 点击“地址”图标 → 跳转至地址管理页（AddressActivity）。
- 点击“客服”图标 → 跳转至客服对话页（ChatTab）。
- 点击订单状态项（如待付款） → 跳转至订单列表页（OrderListActivity）。
- 点击“钱包”、“服务家”任意模块 → 展示Toast提示。
- 所有数据静态读取自 `assets/data/me_tab.json` 文件，无需网络交互。

### 组件说明
1. TopBar：固定在顶部，显示“消息”标题与右侧功能图标。 
2. MessageTabs：用于在不同类型的消息间切换，仅需UI切换状态，无需动态内容。 
3. SubTabs：展示消息子类型（全部、购物、秒送、外卖），点击切换不同静态列表。 
4. MessageList：使用懒加载列表展示最近消息。每条消息为静态内容，不需动态更新。 
5. BottomNav：底部导航栏，维持与其他Tab页面一致的风格与功能。

## 设计规范

### 一、整体风格
- **主色调**：`#E2231A`（京东红）
- **背景色**：`#FFFFFF`（白色）、`#F5F5F5`（底部分区）
- **文字颜色**：
  - 主文字：`#333333`
  - 副文字：`#666666`
  - 标签与说明文字：`#999999`
- **字体规范**：
  - 标题：18sp Bold
  - 模块标题：16sp Medium
  - 普通文字：14sp Regular
- **图标风格**：
  - 统一使用 Material Icons 或 emoji（例如👤、📍、⚙️）。
- **圆角与阴影**：
  - 卡片圆角：10dp
  - 投影高度：`elevation = 2dp`

---

### 二、布局规范
- 顶部信息区固定在最上方，背景使用红色渐变。
- 其他模块通过 `Column` 纵向排列，整体可滚动。
- 模块间距：12dp；左右内边距：16dp。
- 图标大小：24dp；头像大小：72dp。
- 分割线颜色：`#EEEEEE`。
- 列表滚动使用 `LazyColumn` 实现。

---

### 三、交互规范
- 点击头像 → 弹出Toast “个人信息页待开发”。
- 点击“设置” → 跳转至设置页面（SettingActivity）。
- 点击“地址” → 跳转至地址管理页面。
- 点击“待付款”等订单状态 → 跳转至订单列表。
- 点击“更多”按钮 → 弹出提示 “功能开发中”。
- 底部导航栏与其他页面一致，当前页“我的”高亮。

---

### 四、资源与文件命名规范
- **图片资源**：
  - 示例：`icon_me_avatar.png`, `icon_me_wallet.png`, `icon_me_order.png`
  - 存放路径：`app/src/main/res/drawable/`
- **组件文件命名**：
  - `me_header.kt`
  - `me_order_section.kt`
  - `me_tab_page.kt`
- **数据文件命名**：
  - `me_tab.json` 存放静态展示数据。

---

### 五、开发建议
- 架构模式保持一致（MVP或MVVM均可）。
- 页面状态由 `ViewModel` 管理。
- 数据来源统一读取 `assets/data/me_tab.json`。
- 推荐使用 `Card`、`Row`、`Column` 等Compose组件封装模块。
- 避免嵌套滚动，确保滑动性能流畅。
