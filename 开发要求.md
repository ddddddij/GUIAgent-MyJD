# 开发要求
## 任务介绍
我在做安卓开发，并研究如何利用AI来复刻公开的京东APP，但我的复刻无需100%相似，我只需要做到如下几点：
- 1、UI结构基本相似，图标做不到一模一样不要紧，类似即可。
- 2、不需要做出APP的所有页面，例如京东APP，我们只做app需实现的任务用到的典型页面就可以了，其他的页面，比如小视频、秒送页面，我们全部设置为空页面，只保留一个入口给后续开发接入，暂时不需要处理。
- 3、不需要实现动态，例如不需要实现动态数据，商品列表、店铺信息等全部固定死。
- 4、不需要数据库，但是需要保存重要的信号（这些信号我会指定），如购物车商品、订单状态
- 5、不需要登录和联网，仅为本地演示使用
- 6、开发过程，可以考虑截图给AI，让AI实现开发任务。
- 7、没有任何限制，对框架等的选取完全取决于开发者习惯
- 8、不需要通知权限等复杂的东西，至多需要下保存图片到本地、读取相册这样的权限

## 任务要求
1. 开发的时候可以采取逐步开发的策略，慢慢补充需求，慢慢新增数据字段等
2. 避免单个文件代码过长，**必须要遵循MVP模式**
3. 按照指定的页面结构、参考给定的截图作为UI设计指导
4. 严格按照已经定义好的数据结构`app\src\main\java\com\example\MyJD\model`和已有的数据`app\src\main\assets\data`
5. 按照用户要求进行开发，除非用户指定，否则不要引入新的内容。
    - 你只需要参考我给你的截图，**不要脑补**截图中各个按钮点击后会达到的效果。
    - ***截图里面的信息（例如人的名字、人的头像）不必参考***，因为我们自己预设了已有的数据`app\src\main\assets\data`
6. UI中，各个图标若找不到合适的，可以用emoji表情代替。但优先考虑`androidx.compose.material.icons.Icons, androidx.compose.material.icons.filled.AccountCircle`等
7. 严禁执行删除文件等危险操作。
8. 每当你修改完代码，记得更新`README.md`文件，并且确定build正常了再交付给我。禁止修改`开发要求.md`.
9. 如果你需要图片支持，请给我反馈，告诉我需要什么图片。如果你使用了我给你的图片，记得把图片名改为一个可读的名字，便于后期维护。

## 你当前的任务：开发HuaweiP60DetailActivity页面

### 1. 页面功能说明
1. **页面入口**：从首页点击“华为P60”商品栏进入。
2. **展示内容**：
   - 商品主图：使用“app/src/main/assets/image/华为P60封面.JPG”
   - 商品名称：华为P60 双向北斗卫星通信旗舰国行版
   - 商品价格：¥1788
   - 商品规格：8GB+256GB
   - 简要卖点：鸿蒙系统、卫星通信、OLED曲面屏、4800万像素
   - 底部操作栏包含两个按钮：“加入购物车”和“立即购买”
3. **交互功能**：
   - 点击“加入购物车”：保留接口（待后续扩展）
   - 点击“立即购买”：保留接口（待后续扩展）

---

### 2. 页面UI设计规范
| 元素 | 类型 | 样式说明 |
|------|------|----------|
| 页面背景 | View | 背景色 #FFFFFF |
| 商品主图 | ImageView | 宽高比 1:1，居中展示，支持圆角 |
| 商品名称 | TextView | 字体大小 18sp，加粗，颜色 #333333 |
| 商品价格 | TextView | 字体大小 20sp，颜色 #FF0000 |
| 商品规格 | TextView | 字体大小 14sp，颜色 #666666 |
| 卖点标签 | TextView | 横向 LinearLayout 布局，圆角灰底背景 |
| 底部操作栏 | LinearLayout | 高度约 56dp，固定底部 |
| 加入购物车按钮 | Button | 背景色 #F8F8F8，文字颜色 #333333 |
| 立即购买按钮 | Button | 背景色 #FF0000，文字颜色 #FFFFFF |

---

### 4. 数据交互要求
1. 从 `HomeTab` 点击商品卡片时，通过 `Intent` 传递商品ID或名称；
2. `HuaweiP60DetailActivity` 根据接收到的商品信息，加载固定的华为P60数据；

| 滑动页面 | 内容可上下滚动，底部操作栏固定 |

### 6. 技术与开发要求
1. 遵循 **MVP 架构模式**；
2. 新建页面 `HuaweiP60DetailActivity`；
3. 对应布局文件命名为 `activity_huawei_p60_detail.xml`；
4. 商品主图命名为 `huawei_p60_main.png`，放置于 `res/drawable`；
5. 页面数据结构参照 `ProductDetailActivity`；
6. 禁止修改现有的 `ProductDetailActivity`；
7. 若需要复用UI组件（如价格展示区、底部按钮栏），可封装为独立布局；
8. 新增页面完成后更新 `README.md` 文件，确保 Build 正常；
9. 不修改 `开发要求.md` 文件；

---

### 7. 页面跳转逻辑更新
在 `HomeTab` 的商品点击事件中：
- 若商品名称 == “iPhone15” → 跳转至 `ProductDetailActivity`
- 若商品名称 == “华为P60” → 跳转至 `HuaweiP60DetailActivity`
- 其他商品 → 可保留空页面（待后续扩展）
