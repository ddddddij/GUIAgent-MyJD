# 开发要求
## 任务介绍
我在做安卓开发，并研究如何利用AI来复刻公开的京东APP，但我的复刻无需100%相似，我只需要做到如下几点：
- 1、UI结构基本相似，图标做不到一模一样不要紧，类似即可。
- 2、不需要做出APP的所有页面，例如京东APP，我们只做app需实现的任务用到的典型页面就可以了，其他的页面，比如小视频、秒送页面，我们全部设置为空页面，只保留一个入口给后续开发接入，暂时不需要处理。
- 3、不需要实现动态，例如不需要实现动态数据，商品列表、店铺信息等全部固定死。
- 4、不需要数据库，但是需要保存重要的信号（这些信号我会指定），如购物车商品、订单状态
- 5、不需要登录和联网，仅为本地演示使用
- 6、开发过程，可以考虑截图给AI，让AI实现开发任务。
- 7、没有任何限制，对框架等的选取完全取决于开发者习惯
- 8、不需要通知权限等复杂的东西，至多需要下保存图片到本地、读取相册这样的权限

## 任务要求
1. 开发的时候可以采取逐步开发的策略，慢慢补充需求，慢慢新增数据字段等
2. 避免单个文件代码过长，**必须要遵循MVP模式**
3. 按照指定的页面结构、参考给定的截图作为UI设计指导
4. 严格按照已经定义好的数据结构`app\src\main\java\com\example\MyJD\model`和已有的数据`app\src\main\assets\data`
5. 按照用户要求进行开发，除非用户指定，否则不要引入新的内容。
    - 你只需要参考我给你的截图，**不要脑补**截图中各个按钮点击后会达到的效果。
    - ***截图里面的信息（例如人的名字、人的头像）不必参考***，因为我们自己预设了已有的数据`app\src\main\assets\data`
6. UI中，各个图标若找不到合适的，可以用emoji表情代替。但优先考虑`androidx.compose.material.icons.Icons, androidx.compose.material.icons.filled.AccountCircle`等
7. 严禁执行删除文件等危险操作。
8. 每当你修改完代码，记得更新`README.md`文件，并且确定build正常了再交付给我。禁止修改`需求.md`.
9. 如果你需要图片支持，请给我反馈，告诉我需要什么图片。如果你使用了我给你的图片，记得把图片名改为一个可读的名字，便于后期维护。

## AddressListActivity 与 AddressDetailActivity 页面开发要求（收货地址管理）

### 一、你当前的任务
实现 **收货地址管理功能模块**，包括：
- 地址列表页面（AddressListActivity）
- 新增/编辑地址页面（AddressDetailActivity）

用户可从 **MeTab 页面** 进入地址列表页；  
结算页面需支持切换收货地址，默认展示默认地址；  
用户可在列表页中进行 **新增、修改、删除、设为默认、复制** 操作；  
系统中仅允许存在一个默认地址。

---

### 二、页面功能说明

#### （1）AddressListActivity（收货地址列表页）
1. **页面入口**
    - 从 MeTab 页面进入「收货地址管理」。
    - 从结算页点击「更换地址」也可进入该页面。
2. **地址展示列表**
    - 使用 RecyclerView 实现；
    - 每项包含以下内容：
        - 收货人姓名；
        - 手机号；
        - 地址详情（省市区 + 详细地址）；
        - 标签（如 家 / 公司 / 学校 / 默认）；
        - 操作按钮：「删除」「复制」「修改」；
        - 复选框：设置/取消默认地址；
    - 默认地址需在条目中显著标识（红色“默认”标签），且仅允许一个。
3. **交互逻辑**
    - 点击“新增收货地址”按钮 → 跳转至 AddressDetailActivity；
    - 点击“修改” → 跳转至 AddressDetailActivity 并加载该地址详情；
    - 点击“删除” → 弹出确认框后删除该地址；
    - 点击“设为默认” → 自动取消其他地址的默认状态；
    - 点击地址条目（非按钮区域） → 返回结算页并传递该地址信息；
    - 点击“复制” → 将地址信息复制至剪贴板并提示“复制成功”。
4. **返回逻辑**
    - 从结算页进入后选择地址返回时，需将选择的地址信息传回结算页更新展示。

---

#### （2）AddressDetailActivity（新增/编辑收货地址页）
1. **页面入口**
    - 从地址列表页点击“新增收货地址”进入；
    - 或点击“修改”进入并加载已保存的地址信息。
2. **页面内容结构**
    - **收货人信息区域**
        - 输入框：收货人姓名；
        - 输入框：手机号（带区号选择 +86，下拉不可编辑）；
        - 通讯录图标（点击无实际功能）。
    - **地址选择区域**
        - 选项卡切换：「地图选址」与「地区选址」；
        - 显示静态地图占位图；
        - 输入框：地址 + 详细门牌号；
        - 按钮：“地址粘贴板”（无实际功能，仅展示交互）。
    - **设为默认地址开关**
        - Switch 控件，打开则设为默认；
        - 若开启默认，则自动取消其他默认地址。
    - **标签选择区域**
        - 选项包括：学校 / 家 / 公司 / 购物 / 外卖 / 自定义；
        - 单选效果，选中状态以红色边框与字体高亮显示。
    - **确认按钮**
        - 固定底部红色按钮“确认”；
        - 点击后保存地址信息并返回列表页；
        - 若为修改操作则更新原数据。
3. **数据逻辑**
    - 地址信息以本地 SQLite 或 SharedPreferences 存储；
    - 每条地址包含以下字段：
      | 字段名 | 类型 | 说明 |
      |--------|------|------|
      | id | Int | 地址唯一标识 |
      | name | String | 收货人姓名 |
      | phone | String | 手机号码 |
      | area | String | 所在省市区 |
      | detail | String | 详细门牌号 |
      | tag | String | 地址标签 |
      | isDefault | Boolean | 是否默认地址 |
    - 若设为默认地址，则更新所有其他记录的 isDefault=false。

### 五、功能补充说明
1. 地址数据在应用退出后需持久化存储；
2. 结算页面进入时若存在默认地址，优先显示默认；
3. 切换默认地址后更新所有模块引用；
4. 若无地址数据，则显示“暂无地址，请新增收货地址”空态页；
5. 所有表单输入框需进行基本校验（姓名非空、手机号格式正确、地址非空）；
6. 地址数据改变后列表也要同步更新。
