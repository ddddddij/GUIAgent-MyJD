# 开发要求
## 任务介绍
我在做安卓开发，并研究如何利用AI来复刻公开的京东APP，但我的复刻无需100%相似，我只需要做到如下几点：
- 1、UI结构基本相似，图标做不到一模一样不要紧，类似即可。
- 2、不需要做出APP的所有页面，例如京东APP，我们只做app需实现的任务用到的典型页面就可以了，其他的页面，比如小视频、秒送页面，我们全部设置为空页面，只保留一个入口给后续开发接入，暂时不需要处理。
- 3、不需要实现动态，例如不需要实现动态数据，商品列表、店铺信息等全部固定死。
- 4、不需要数据库，但是需要保存重要的信号（这些信号我会指定），如购物车商品、订单状态
- 5、不需要登录和联网，仅为本地演示使用
- 6、开发过程，可以考虑截图给AI，让AI实现开发任务。
- 7、没有任何限制，对框架等的选取完全取决于开发者习惯
- 8、不需要通知权限等复杂的东西，至多需要下保存图片到本地、读取相册这样的权限

## 任务要求
1. 开发的时候可以采取逐步开发的策略，慢慢补充需求，慢慢新增数据字段等
2. 避免单个文件代码过长，**必须要遵循MVP模式**
3. 按照指定的页面结构、参考给定的截图作为UI设计指导
4. 严格按照已经定义好的数据结构`app\src\main\java\com\example\MyJD\model`和已有的数据`app\src\main\assets\data`
5. 按照用户要求进行开发，除非用户指定，否则不要引入新的内容。
    - 你只需要参考我给你的截图，**不要脑补**截图中各个按钮点击后会达到的效果。
    - ***截图里面的信息（例如人的名字、人的头像）不必参考***，因为我们自己预设了已有的数据`app\src\main\assets\data`
6. UI中，各个图标若找不到合适的，可以用emoji表情代替。但优先考虑`androidx.compose.material.icons.Icons, androidx.compose.material.icons.filled.AccountCircle`等
7. 严禁执行删除文件等危险操作。
8. 每当你修改完代码，记得更新`README.md`文件，并且确定build正常了再交付给我。禁止修改`需求.md`.
9. 如果你需要图片支持，请给我反馈，告诉我需要什么图片。如果你使用了我给你的图片，记得把图片名改为一个可读的名字，便于后期维护。

## ShopPageActivity 页面开发要求（店铺主页）

### 一、你当前的任务
实现 **Apple 产品京东自营旗舰店** 的模拟主页。该页面为从“消息设置页”点击“进入店铺”后跳转进入的新页面，主要展示店铺信息、商品列表及商品详情入口。页面需根据参考截图（见 `店铺主页参考.jpg`）实现静态布局，商品数据固定写入本地 assets 文件中。用户可从本页点击商品进入商品详情页，复用已有购物车、购买、订单、结算功能逻辑。

---

### 二、页面功能说明
1. **页面入口**  
   从消息设置页点击“进入店铺”进入本页。无需网络请求，数据从本地静态文件中加载（或直接硬编码）。
2. **页面头部区域**
   - 显示店铺头像、店铺名称（Apple 产品京东自营旗舰店）、粉丝数。
   - 右上角展示“已关注”状态（不可切换）。
   - 下方展示“Apple 全方位服务，尽在京东”横幅说明。
3. **店铺统计信息栏**
   - 展示若干统计项（近30天新增关注、满意度等），仅作静态展示。
4. **商品分类标签**
   - 标签栏包括：**推荐**、**销量**、**新品**、**价格**、**仅看有货**；
   - 点击无实际功能，仅显示选中状态样式。
5. **商品列表区域**
   - 使用 RecyclerView 实现；
   - 每个商品卡片包括：
      - 商品图片；
      - 商品标题；
      - 商品价格（红色字体显示当前价，灰色显示原价）；
      - 商品评价信息（静态展示）；
      - 右侧显示“+”按钮（点击可加入购物车）。
   - 商品列表固定 4~5 条数据（如 iPhone 15、iPhone 16 Pro、iPhone 17 等）。
6. **页面跳转逻辑**
   - 点击商品卡片 → 进入 `ProductDetailActivity`；
   - 点击“+”按钮 → 调用已有购物车逻辑，加入购物车；
   - 点击返回按钮 → 返回消息设置页。

### 五、交互要求
1. **点击返回按钮**：返回消息设置页。
2. **点击商品卡片**：进入 `ProductDetailActivity`，复用现有商品详情页逻辑。
3. **点击“+”按钮**：调用已有购物车逻辑，将商品添加至购物车。
4. **点击分类标签**：切换标签选中状态（UI变化），无过滤逻辑。
5. **页面滚动**：支持纵向滑动查看商品。

### 六、开发要点
1. **数据加载**
   - 从本地 assets 或 model 层读取固定数据，例如：
     ```json
     {
       "shop_name": "Apple产品京东自营旗舰店",
       "fans": "9090.3万",
       "products": [
         {"name": "Apple/苹果 iPhone 15 (A3092) 128GB 蓝色", "price": 3799, "origin_price": 3899, "img": "iphone15_blue.jpg"},
         {"name": "Apple/苹果 iPhone 16 Pro (A3294) 128GB 白色钛金属", "price": 6199, "origin_price": 6399, "img": "iphone16pro_white.jpg"},
         {"name": "Apple/苹果 iPhone 17 256GB 薰衣草紫色", "price": 5999, "origin_price": 6299, "img": "iphone17_purple.jpg"}
       ]
     }
     ```
2. **图片资源管理**
   - 所有图片命名为可读格式（如 `iphone15_blue.jpg`）；
   - 图片路径放入 `app/src/main/res/drawable` 或 `assets/images`；
   - 若找不到图片，使用占位图。
3. **代码结构**
   - 严格遵循 MVP 模式：
      - **View 层**：`ShopPageActivity` + `ShopAdapter`；
      - **Presenter 层**：负责加载数据与事件逻辑；
      - **Model 层**：定义商品与店铺数据结构。
4. **复用逻辑**
   - 商品点击 → 调用现有 `ProductDetailActivity`；
   - 加入购物车 → 调用已有购物车逻辑；
   - 购买、结算、订单流程 → 全部复用现有模块。
5. **UI 适配**
   - 适配不同屏幕密度；
   - 保证商品卡片对齐、美观；
   - 分隔线使用颜色 #EEEEEE。
